# 进度快照 018 - 最小化到托盘图标缺失修复完成

## 本次已完成

1. 完成托盘图标路径修复
   - 修复 Windows 托盘初始化时错误使用 `Platform.resolvedExecutable` 作为图标路径的问题。
   - 改为使用真实资源路径：
     - `assets/app_icon.ico`
   - 保持 Linux 与 macOS 原有分支逻辑不变。

2. 完成托盘初始化结果可感知改造
   - `initSystemTray()` 已改为返回 `Future<bool>`。
   - 现在会显式检查 `systemTray.initSystemTray(...)` 的返回值。
   - 初始化失败时会写入启动诊断日志并输出调试日志，避免“窗口隐藏了但托盘没入口”时无迹可查。

3. 完成最小化到托盘时序修复
   - `MainApp` 中新增托盘初始化 Future 持有。
   - 关闭行为为“最小化到托盘”时，会先等待托盘初始化完成。
   - 如果托盘初始化失败，当前窗口不会再被静默隐藏。

4. 完成托盘未初始化时的安全兜底
   - `SystemTrayManager.updateMenu()`
   - `SystemTrayManager.updateTooltip()`
   - 上述方法在托盘实例未初始化时会安全跳过，不再抛出未初始化异常。

5. 完成托盘图标路径单测补充
   - 新增纯 Dart 工具函数与测试，覆盖 Windows 与非 Windows 平台的托盘图标资源选择逻辑。
