# 010 远程协助与客户端生命周期绑定完成

## 本轮目标

- 让 `vntcrustdesk` 与 `VNTC` 客户端形成绑定运行/绑定停止关系
- 避免出现：
  - 客户端已运行，但 `vntcrustdesk` 未运行
  - 客户端已退出，但 `vntcrustdesk` 仍然残留运行
- 按客户端真实启动与真实关闭路径完成验证

## 已完成内容

### 1. 已完成本轮备份

- 已创建备份：
  - `backup\015_远程协助生命周期绑定修复前备份_20260601_112642`

### 2. 已完成远程协助退出收口能力

- 已新增：
  - `lib/remote_assist/remote_assist_health_service.dart`
    - `shutdownBackgroundSilently()`
- 当前客户端退出时会按顺序执行：
  1. 查询服务与后台进程状态
  2. 优先尝试 `Stop-Service`
  3. 若仍有残留进程，再强制清理 `vntcrustdesk` 进程
  4. 再次复查服务状态与进程数量
- 已补充完整诊断日志，便于定位退出时是否成功收口

### 3. 已完成远程协助管理器生命周期防竞态

- 已修改：
  - `lib/remote_assist/remote_assist_manager.dart`
- 新增：
  - `_stopping` 状态位
- 已修复的关键问题：
  - 当客户端正在退出时，后台刷新线程不会再继续执行“静默拉起远程协助”
  - 避免出现“应用刚退出，后台又把 `vntcrustdesk` 重新拉起来”的竞态
- `stop()` 现在会：
  - 停止定时刷新
  - 停止 Presence
  - 调用 `shutdownBackgroundSilently()`
  - 再清空状态并通知界面

### 4. 已补齐托盘退出路径

- 已修改：
  - `lib/system_tray_manager.dart`
- 托盘菜单“退出”现在不再直接 `exit(0)`
- 现在会先执行：
  - `RemoteAssistManager.instance.stop()`
  - `vntManager.removeAll()`
- 这样托盘退出也会同步停止 `vntcrustdesk`

### 5. 已补齐后续安装/修复权限规则

- 已修改：
  - `scripts/bootstrap_vntcrustdesk.ps1`
- 原先只保证普通交互用户可以“启动服务”
- 现在已升级为同时保证：
  - 可启动
  - 可停止
- 服务描述符目标 ACE 已调整为：
  - `(A;;CCLCSWRPWPLOCRRC;;;IU)`
- 这样后续新装或修复后的 `vntcrustdesk` 会更稳定地支持客户端退出时停服

### 6. 已完成真实启动/真实关闭路径验证

- 已重新导出便携包：
  - `dist\portable\VNT_App_2.2_Windows_Portable.zip`
- 已按真实路径验证：
  1. 停止 `vntcrustdesk` 服务与进程
  2. 启动便携版：
     - `dist\portable\VNT_App_2.2_Windows_Portable\vnt_app.exe`
  3. 等待客户端静默拉起远程协助后台
  4. 通过窗口标准 `WM_CLOSE` 关闭应用，让它走真实退出链路

- 实测结果：
  - 关闭前：
    - `BEFORE_SERVICE=Running`
    - `BEFORE_PROCESS_COUNT=2`
    - `BEFORE_LISTEN=YES`
  - 关闭后：
    - `APP_EXITED=True`
    - `AFTER_SERVICE=Stopped`
    - `AFTER_PROCESS_COUNT=0`
    - `AFTER_LISTEN=NO`

- 说明已经实现：
  - 客户端启动后，`vntcrustdesk` 会被同步拉起
  - 客户端正常退出后，`vntcrustdesk` 会被同步停止
