# 013 远程密码配置链路源码接入完成

## 本轮目标

- 在主项目远程协助页增加“设置远程密码”入口
- 打通主项目到 `vntcrustdesk` 的外部无界面密码配置命令
- 去掉 `vntcrustdesk` 自身桌面/移动端密码弹窗的复杂度限制

## 已完成内容

### 1. 已完成本轮备份

- 已创建备份：
  - `backup\018_远程密码配置链路前备份_20260601_124954`
- 已按规则继续只保留最近 10 份备份

### 2. 已完成主项目密码配置入口

- 远程协助页顶部已新增文字按钮：
  - `设置远程密码`
- 位置已放在：
  - `刷新` 按钮左侧
- 点击后会弹窗显示：
  - 单个密码输入框
  - “留空即清空密码并恢复手动接受”的说明

### 3. 已完成主项目配置动作接入

- 已新增：
  - `RemoteAssistManager.configureAccessPassword(String password)`
  - `RemoteAssistLauncher.configureAccessPassword(String password)`
- 当前逻辑：
  - 先确保 `vntcrustdesk` 后台已可用
  - 再调用 `vntcrustdesk.exe --configure-access-password <password>`
  - 成功后刷新远程协助状态
- 当前 UI 反馈：
  - 非空密码：提示已启用无人值守密码
  - 空密码：提示已清空密码并恢复手动接受

### 4. 已完成 vntcrustdesk 外部无界面配置入口

- `vntcrustdesk-src` 已新增启动参数处理：
  - `--configure-access-password`
- 该命令会在不进入主界面流程前直接执行：
  - 设置永久密码
  - 设置 `verification-method = use-permanent-password`
  - 设置 `approve-mode = password / click`
- 规则已固定：
  - 密码非空：
    - `approve-mode = password`
  - 密码为空：
    - `approve-mode = click`
- 已加入写入后读取校验，失败时返回非 `0` 退出码

### 5. 已完成自身密码弹窗限制收敛

- `vntcrustdesk` 桌面端“Set Password”弹窗已移除：
  - 8 位限制
  - 大写字母限制
  - 小写字母限制
  - 数字限制
  - 强度提示区块
- 仍保留：
  - 最大长度由底层 `mainMaxEncryptLen()` 控制
  - 二次确认必须一致
- `vntcrustdesk` 移动端“Set your own password”弹窗已移除：
  - “至少 6 位”限制
- 移动端同样保留：
  - 确认输入一致校验
