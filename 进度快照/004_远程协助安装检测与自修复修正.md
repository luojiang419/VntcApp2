# 004 远程协助安装检测与自修复修正

## 本轮目标

修复“远程协助页面始终提示未检测到 `vntcrustdesk` 安装”的问题，并重新导出可给用户测试的便携包与安装包。

## 已完成

### 1. 运行时检测修正

- `RemoteAssistLauncher` 不再只依赖 `remote_assist/vntcrustdesk_manifest.json`
- 当 manifest 缺失时，新增通过以下信息继续探测安装：
  - Windows 服务 `vntcrustdesk`
  - 卸载注册表中的 `VNTC RustDesk` / `vntcrustdesk`
- 因此安装阶段若 bootstrap 未成功写入 manifest，页面也不会再直接误判为“未安装”

### 2. 应用内自修复补齐

- 远程协助页“修复”按钮升级为“安装/修复”
- 若当前运行目录携带：
  - `remote_assist/artifacts/vntcrustdesk.msi`
  - `scripts/bootstrap_vntcrustdesk.ps1`
- 则点击按钮后会触发管理员提权，自动执行 `vntcrustdesk` 安装/修复
- 若用户取消 UAC，会给出明确错误提示
- 若运行目录没有带安装组件，会明确提示“当前运行目录未携带远程协助安装组件”

### 3. 页面状态文案修正

- 新增“安装方式”更细粒度描述：
  - `受当前安装器管理`
  - `已检测到独立安装（未绑定当前应用）`
  - `当前目录已携带远程协助安装组件`
  - `当前目录未携带远程协助安装组件`
- 新增“修复能力”描述，区分：
  - 可自动安装并修复
  - 仅能检查现有安装
  - 仅能检测，无法补装

### 4. 便携包资源补齐

- `scripts/export_portable_package.ps1` 已修改
- 新版便携包现在会一并带上：
  - `remote_assist/artifacts/vntcrustdesk.msi`
  - `scripts/bootstrap_vntcrustdesk.ps1`
  - `scripts/uninstall_vntcrustdesk.ps1`
- 这样用户运行便携包时，也可以在应用内完成远程协助组件安装/修复

### 5. 新产物已重新导出

- 便携包：
  - `dist/portable/VNT_App_2.0_Windows_Portable.zip`
- 安装包：
  - `dist/installer/VNT_App_2.0_Windows_Setup.exe`
- 本轮导出时已确认便携包内存在 `vntcrustdesk.msi` 和 bootstrap/uninstall 脚本

## 本轮验证

- 已通过：修改后的 PowerShell 脚本语法解析
- 已通过：针对本次修改文件的 `dart analyze` 错误级检查
- 已补充：`test/remote_assist_health_status_test.dart`
- 已成功：重新执行 `scripts/export_installer_package.ps1`，完整导出便携包与安装包

## 仍待用户实测

- 安装版首次进入“远程协助”页面后，点击“安装/修复”是否能正常拉起 UAC 并完成补装
- 便携版首次进入“远程协助”页面后，点击“安装/修复”是否能正常安装 `vntcrustdesk`
- 修复后页面是否变为：
  - `vntcrustdesk 已安装`
  - `服务运行中`
  - `49999 监听中`
- 双机 VNT 虚拟 IP 直连是否能正常进入受控确认流程
