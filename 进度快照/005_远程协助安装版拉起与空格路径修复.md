# 005 远程协助安装版拉起与空格路径修复

## 本轮目标

修复安装版 `VNT App 2.0` 安装完成后 `vntcrustdesk` 未自动装起、远程协助页无法正常工作的真实安装链路问题，并完成首次安装与应用内修复两条链路的端到端验证。

## 已完成内容

### 1. 根因定位完成

- 已按真实用户链路复现安装版问题：
  - 清理当前机器上的 `VNT App 2.0` 与 `VNTC RustDesk`
  - 重新运行 `dist/installer/VNT_App_2.0_Windows_Setup.exe`
  - 读取安装日志 `output/install_vnt_app_setup.log`
- 确认安装器确实执行了 `bootstrap_vntcrustdesk.ps1`
- 确认失败点不是 MSI 本体，而是安装脚本对带空格路径的 `msiexec` 参数传递错误
- 在 `C:\Program Files\VNT App 2.0` 场景下，旧脚本会报：
  - `vntcrustdesk MSI install failed: 1639`
- 结论：
  - 便携目录无空格，所以之前便携验证能过
  - 安装版位于 `Program Files`，路径含空格，导致安装链路失效

### 2. 代码修复完成

- `scripts/bootstrap_vntcrustdesk.ps1`
  - 新增 `Quote-ProcessArgument`
  - 修正 `msiexec /i` 时对 `MSI` 路径和安装日志路径的传参
- `scripts/uninstall_vntcrustdesk.ps1`
  - 新增 `Ensure-Directory`
  - 新增 `Quote-ProcessArgument`
  - 修正 `msiexec /x` 时对卸载日志路径的传参
- `lib/remote_assist/remote_assist_launcher.dart`
  - 修正应用内“安装/修复”提权脚本调用时，`-File / -AppDir / -MsiPath` 对带空格路径的参数传递

### 3. 新产物重新导出完成

- 已重新执行：
  - `scripts/export_installer_package.ps1`
- 新安装包：
  - `dist/installer/VNT_App_2.0_Windows_Setup.exe`
- 新便携包：
